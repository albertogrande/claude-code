## Verdict

The architecture is genuinely good: a clean three-layer design (frontmatter-driven Astro
site → machine JSON endpoints → thin stateless MCP proxy), strict zod schemas so agents can
write deterministically, an eval harness that exists at all (rare), and a scout → weekly →
deep-dive editorial separation with memory. The weaknesses are not in the design but in the
**connective tissue**: the quality gates don't actually run on the content that matters, the
pipeline fails silently, and the highest-value content (practices) is invisible to humans.

## P0 — Reliability: the gates don't gate

1. **Editorial commits bypass CI entirely.** Writers push to `main` with `github.token`;
   GitHub's loop-prevention means those pushes never fire the `push:` event, so `ci.yml`
   (build check + MCP retrieval gate) never runs on scout/weekly/deep-dive/probes commits.
   The only build they get is Deploy's own — and if that fails, the site silently stays stale.
   *Fix:* writers open a PR (CI runs, auto-merge on green) or push with a PAT.

2. **No failure notifications anywhere.** None of the five workflows has an `if: failure()`
   step. Token expiry, timeouts, rejected pushes, broken Deploy builds — all die silently in
   the Actions tab. A crashed scout is indistinguishable from a quiet day (the `(quiet day)`
   marker is only written when the job *runs*). *Fix:* failure step that opens an issue.

3. **Probes never publishes.** `deploy.yml:11` lists Scout/Weekly/Deep Dive but omits
   `"Probes"`, so probe stamps/retirements to `src/content/practices/` never deploy — and the
   next probe run measures the stale production corpus (feedback loop). Also: the probes
   regression gate runs *inside the Claude prompt* (`probes.yml:52`), so a failing eval can't
   fail the workflow. *Fix:* add `"Probes"` to Deploy; make the eval a real gated job step;
   guard Deploy on `workflow_run.conclusion == 'success'`.

4. **Writers can silently lose a whole run.** No `concurrency:` group and no fetch/rebase
   before push in scout/weekly/deep-dive/probes — a manual dispatch overlapping a scheduled
   run gets a non-fast-forward reject and the run's work is discarded. Also `weekly.yml`
   gives 30 min to digest + guide pass + practices + (sometimes) a full deep dive; on timeout
   the commit step never runs and everything is lost. *Fix:* shared `concurrency` group,
   rebase-then-push, raise the weekly timeout and/or dispatch the commissioned dive as its
   own job.

5. **MCP has two hard-failure modes on upstream blips.** `mcp/lib.js:25` fetches with no
   `AbortSignal` timeout (a slow GitHub Pages hangs every tool until the platform kills it),
   and `getJson` (`lib.js:20-30`) throws once the 5-min TTL expires instead of serving the
   last-known-good value. *Fix:* `AbortSignal.timeout(...)` + stale-on-error fallback.

6. **The eval gate doesn't protect the property the search fix was for.**
   `mcp/eval/run-evals.mjs:101-105` gates only in-domain hit@3. Off-topic noise (the exact
   bug the stopword fix addressed) and Spanish recall are measured but never fail CI.
   *Fix:* add an off-topic clean-rate threshold and a Spanish hit@3 threshold to the gate.

## P1 — Product: activate the data the schemas already carry

7. **Practices have no human surface.** `content.config.ts:55` promises "a human-readable
   page"; none exists. The 11 practices — the most granular, highest-value content — render
   nowhere for humans, aren't in the nav, the ⌘K palette, or the Pagefind index.
   *Fix:* `/practices` page (+ per-practice anchors), `data-pagefind-body`, nav entry.

8. **Tags are dead ends; cross-linking is one-directional.** Tag chips aren't links (no
   `/tags/[tag]`), and guide sections have no backlinks to the practices/weeklies/dives that
   reference them, even though `practice.section` and `related[]` carry the data.
   *Fix:* tag index pages + "practices for this section / mentioned in" backlink blocks.

9. **No referential integrity at build.** `related[].href` and `practice.section` are free
   strings; a typo ships as a silently broken link. Dangerous when the authors are agents.
   *Fix:* build-time check that every `section`/internal `related` href resolves.

10. **SEO/social pass missing:** no `og:image` (bare social shares, `twitter:card=summary`),
    `og:type=website` even on articles, no `robots.txt` (no `public/` at all → no favicon),
    no `404.astro` (GitHub's generic 404 loses site chrome), no JSON-LD, `<time>` without
    `datetime`, sitemap includes the machine endpoints (no `filter`).

11. **User-facing wrongness in navigation:** homepage advertises `d` (deep dives) and `?`
    (about) shortcuts that don't exist / do something else (`index.astro:115-116` vs
    `Shortcuts.astro:207,212`); the ⌘K palette and help modal omit deep-dives entirely;
    first-time visitors on a dark OS get a light site (boot script defaults `'light'`,
    no `prefers-color-scheme` fallback in `main.scss`).

12. **MCP search functional gaps:** tag-only search is impossible (`query` required; the
    "empty query matches all" branch at `lib.js:95` scores 1 but the threshold at `lib.js:167`
    is ≥2 — dead code); `tags` aren't validated against the enum (a typo returns "no matches"
    with no hint); `since`/`verify` are rendered but not indexed; ranking has no IDF, so
    precision will degrade as the practices corpus grows (which is the roadmap).

## P2 — Tests, docs, hygiene

13. **Tests that assert less than they appear to:** the MCP tag-filter test (`test.mjs:82-83`)
    cannot fail (the no-match message itself matches the regex); the cache is never exercised
    (TTL forced to 0); no error-path test for upstream-down.
14. **Docs drift:** `docs/agent-access.md:57-59` describes the abandoned `server.js` Vercel
    deploy (reality: pinned `api/mcp.js` via `vercel.json`); "always current" claims vs the
    5-min TTL; `Dockerfile` uses `npm install` without the lockfile (non-reproducible);
    `content.config.ts:4` says "three rendered collections" (there are four).
15. **`FABLE5-MISSION.md` is stale and belongs elsewhere:** it points a fresh session at
    `.claude/skills/radar-scan/SKILL.md` and `radar.yml`, which no longer exist, and
    describes this repo pre-refactor. It's a mission brief for a different project sitting
    on `main`. Archive it (or move to `docs/missions/`) and fix the references.
16. **Probe bookkeeping:** the 2026-07-10 baseline claims frontmatter stamping that didn't
    happen (no practice carries `probe:`), and the question bank is missing the three
    practices added 2026-07-10.
17. **Code duplication in the site:** the date-desc / order sorts are copy-pasted across
    ~10 files while `src/lib/site.ts:23-26` defines the intended shared `byDateDesc` that
    nobody imports (and the inline copies lose its id tiebreak); `weekly/[...slug]` and
    `deep-dives/[...slug]` share a ~95% identical foot block that should be one component.
18. **The pipeline contradicts its own guidance:** every workflow runs `claude-sonnet-5`
    while the site's own practice says to switch to Fable for long unattended runs. Either
    switch the long jobs or record the cost rationale.

## Roadmap to state-of-the-art

The repo's product *is* trust: a guide that is never wrong, never silently broken, and
retrievable by humans and agents alike. In order:

- **Phase 1 — Trust the pipeline (P0, items 1-6).** PRs-with-CI for writers, failure
  notifications, Deploy wiring, concurrency, MCP timeout + stale-on-error, hardened eval
  gates. Until this lands, every other improvement can silently rot.
- **Phase 2 — Surface the product (P1, items 7-12).** Practices pages, tag routes,
  backlinks, referential integrity, SEO pass, nav fixes, MCP tag/ranking fixes.
- **Phase 3 — SOTA differentiators.** A fact-integrity gate (source-liveness check + a
  second-model verification pass over changed claims before publish); IDF or embeddings
  retrieval once the practices corpus passes ~30 items; a shared schema contract between
  the site's JSON endpoints and the MCP (today a field rename breaks silently); per-run
  cost/quality ledger; Fable for the long unattended jobs.
